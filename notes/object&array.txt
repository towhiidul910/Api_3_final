# 🧠 JavaScript / TypeScript Cheat Sheet (Array vs Object vs Record)

---

## 📦 ARRAY

### ✅ Definition

Ordered list of values

```js
[1, 2, 3]
["a", "b", "c"]
[{ id: 1 }, { id: 2 }]
```

### ✅ Access

```js
arr[0] // first item
```

### ✅ Use When

* Order matters
* You loop (`map`, `forEach`)
* You render UI lists
* You allow duplicates

---

## 🧱 OBJECT

### ✅ Definition

Key → Value mapping

```js
{
  id: 1,
  name: "Laptop"
}
```

### ✅ Access

```js
obj.id
obj["id"]
```

### ⚠️ Rules

* Keys must be unique
* Duplicate keys get overwritten

```js
{
  id: 1,
  id: 2
}
// result → { id: 2 }
```

### ✅ Use When

* Fast lookup needed
* Mapping IDs → values
* Structured data

---

## ⚔️ ARRAY vs OBJECT

| Feature       | Array 📦           | Object 🧱       |
| ------------- | ------------------ | --------------- |
| Structure     | Ordered list       | Key-value pairs |
| Access        | Index (`arr[0]`)   | Key (`obj.id`)  |
| Order matters | ✅ Yes              | ❌ No            |
| Fast lookup   | ❌ No (loop needed) | ✅ Yes           |

---

## 🧠 TYPESCRIPT: RECORD

### ✅ Definition

Utility type for objects

```ts
Record<KeyType, ValueType>
```

### Example

```ts
type Box = "box-1" | "box-2";

const state: Record<Box, string> = {
  "box-1": "dz-1",
  "box-2": "dz-2"
};
```

### Meaning

* Keys must be `"box-1"` and `"box-2"`
* Values must be `string`

---

## 🔥 RECORD vs OBJECT TYPE

```ts
// Record
Record<"a" | "b", number>

// Same as:
{
  a: number;
  b: number;
}
```

---

## 🎮 DnD MENTAL MODEL

### Object (Best)

```ts
{
  "box-1": "dz-1",
  "box-2": "dz-2"
}
```

👉 Fast: "Where is this box?"

---

### Array (Slower)

```ts
[
  { box: "box-1", location: "dz-1" }
]
```

👉 Need to search every time

---

## 💡 PRO TIPS

* Use **Object / Record** → for lookup (ID → value)
* Use **Array** → for rendering lists
* Real apps use BOTH together

---

## ⚡ ULTRA SHORT TL;DR

* Array = list `[a, b, c]`
* Object = map `{ key: value }`
* Record = typed object (TypeScript)
* Array → order + loops
* Object → fast lookup
