# 🧠 TL;DR

| Method      | Returns New Array? | Works With `await`?          | Best Use Case                              |
| ----------- | ------------------ | ---------------------------- | ------------------------------------------ |
| `forEach()` | ❌ No               | ❌ No                         | Side effects only (logging, pushing, etc.) |
| `map()`     | ✅ Yes              | ⚠️ Only with `Promise.all()` | Transform array into new array             |
| `for...of`  | ❌ No               | ✅ Yes                        | Sequential async operations                |

---

# 1️⃣ `forEach()` — “Just loop”

### What it does:

Runs a function for every item.

```js
const numbers = [1, 2, 3];

numbers.forEach((num) => {
  console.log(num);
});
```

### Key Traits:

* Doesn’t return anything
* Doesn’t wait for async
* Cannot break early
* Cannot use `await` properly

### ❌ Bad for async:

```js
numbers.forEach(async (num) => {
  await something();
});
```

It won’t wait. The outer function continues immediately.

### ✅ Use `forEach()` when:

* Logging
* Mutating existing array
* Pushing into another array manually
* No async involved

---

# 2️⃣ `map()` — “Transform into something new”

### What it does:

Creates a NEW array based on transformation.

```js
const numbers = [1, 2, 3];

const doubled = numbers.map((num) => num * 2);

console.log(doubled); // [2, 4, 6]
```

### Key Traits:

* Returns a new array
* Does NOT modify original
* Perfect for transformation

---

### 🚀 With Async (Correct Way)

```js
await Promise.all(
  numbers.map((num) => asyncOperation(num))
);
```

Why this works:

* `map()` creates array of promises
* `Promise.all()` waits for all

### ✅ Use `map()` when:

* You need to transform data
* You need array output
* You’re preparing data for DB
* You're building Promise arrays

---

# 3️⃣ `for...of` — “Real loop”

This is a classic loop for arrays.

```js
for (const num of numbers) {
  console.log(num);
}
```

### Key Traits:

* Works with `await`
* Can use `break`
* Can use `continue`
* Executes sequentially

---

### ✅ Async-safe example:

```js
for (const num of numbers) {
  await asyncOperation(num);
}
```

This waits for each one before continuing.

---

# 🧠 Mental Model

### `forEach()`

“Run this for each item. I don’t care about return.”

### `map()`

“Transform this into something new.”

### `for...of`

“Loop properly and give me control.”

# 🔥 Backend Use Case Examples

## In Prisma

### ❌ Bad:

```js
files.forEach(async (file) => {
  await prisma.image.create(...)
});
```

### ✅ Sequential:

```js
for (const file of files) {
  await prisma.image.create(...)
}
```

### 🚀 Best Parallel:

```js
await Promise.all(
  files.map(file =>
    prisma.image.create(...)
  )
);
```

---

# 🧩 Performance Difference

| Method            | Execution Style |
| ----------------- | --------------- |
| forEach           | Fire & forget   |
| for...of          | Sequential      |
| map + Promise.all | Parallel        |

Parallel is fastest.

---

# ⚠️ One Important Rule

If you see:

```js
array.forEach(async ...)
```

🚩 That’s almost always wrong in backend code.

---

# 💡 Tips & Tricks

* Use `map()` when building arrays.
* Use `for...of` when order matters.
* Use `Promise.all()` when you want speed.
* Avoid `forEach()` for async DB calls.
* `createMany()` is even better than looping when possible.
