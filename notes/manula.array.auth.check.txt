# 🧠 When Would You Need Manual Ownership Check?

You’d need it when:

### Scenario 1 — You fetched by IDs only

```ts
const images = await prisma.image.findMany({
  where: { id: { in: imgIds } }
});
```

Now Prisma does NOT filter by userId.

So you must manually check:

```ts
for (const img of images) {
  if (img.userId !== userId)
    throw new AppError("Delete Image", "Unauthorized", 403);
}
```

---

### Scenario 2 — You’re using raw SQL

```ts
const images = await prisma.$queryRaw`SELECT * FROM Image WHERE id IN (...)`;
```

Again, no ownership filtering.
Manual check required.

---

### Scenario 3 — Data came from cache or memory

If images came from Redis or some in-memory store,
you must verify ownership manually.

---

# 🧠 Is Looping the Right Way?

Yes — but there are cleaner ways.

Instead of:

```ts
for (const img of images) {
  if (img.userId !== userId)
    throw new AppError(...)
}
```

You can use `.every()`:

```ts
if (!images.every(img => img.userId === userId)) {
  throw new AppError("Delete Image", "Unauthorized", 403);
}
```

That’s cleaner and more expressive.

---

# 🧠 Even Cleaner Pattern

```ts
const unauthorized = images.some(img => img.userId !== userId);

if (unauthorized)
  throw new AppError("Delete Image", "Unauthorized", 403);
```

`some()` is nice because:

* Stops at first failure
* Very readable

---

# 🏆 Best Practice Philosophy

If possible:

✅ Enforce ownership in database query
❗ Only manually check when DB filtering isn’t used

Why?

Because database filtering is:

* Faster
* Cleaner
* Less error-prone
* Scales better

Manual loops are fine — but second-best.

---

# 🧠 Subtle Security Insight

Even if you manually check ownership,
you should still verify count:

```ts
if (images.length !== imgIds.length)
```

Because if attacker sends 5 IDs
and only 3 exist,
manual loop won’t detect missing IDs.

Length comparison catches tampering.

---

# 🛠 Clean Manual Ownership Check Example

```ts
const images = await prisma.image.findMany({
  where: { id: { in: imgIds } }
});

if (images.length !== imgIds.length)
  throw new AppError("Delete Image", "Invalid request", 400);

if (!images.every(img => img.userId === userId))
  throw new AppError("Delete Image", "Unauthorized", 403);
```

That’s the safest manual pattern.

* Know when you must validate manually
